# Mutual Fund FAQ Assistant

> A **facts-only RAG-based chatbot** for mutual fund scheme queries, built using Groww as the reference product context.

![Disclaimer](https://img.shields.io/badge/Facts--only-No%20investment%20advice-orange)

---

## Overview

This assistant answers objective, verifiable queries about 5 mutual fund schemes by retrieving information exclusively from official Groww fund pages. It uses a **Retrieval-Augmented Generation (RAG)** pipeline with BGE embeddings and Groq LLM.

**It strictly does NOT provide investment advice, opinions, or recommendations.**

## Architecture Overview

The system uses a **RAG (Retrieval-Augmented Generation)** approach:

```
User Query → Input Sanitizer → Query Classifier
                                    │
                        ┌───────────┴──────────────┐
                    FACTUAL                    ADVISORY
                        │                          │
              Embed → FAISS Search          Polite Refusal
                        │                   + AMFI Link
                  Context Builder
                        │
                  Prompt Builder → Groq LLM
                        │
               Response Validator → Formatter
                        │
         Answer + Groww Source Link + Last Updated Date
```

For the full architecture, see [docs/architecture.md](docs/architecture.md).

## Setup Instructions

### 1. Prerequisites

- Python 3.10+
- A [Groq API key](https://console.groq.com) (free tier available)

### 2. Clone and set up environment

```bash
git clone <repo-url>
cd mutual-fund-faq

# Create and activate virtual environment
python -m venv venv
venv\Scripts\activate       # Windows
# source venv/bin/activate  # Linux/Mac

# Install dependencies
pip install -r requirements.txt
```

### 3. Configure environment variables

```bash
copy .env.example .env
# Edit .env and add your GROQ_API_KEY
```

### 4. Build the knowledge base (run ingestion)

```bash
python ingestion/ingest.py
```

This scrapes the 5 Groww fund pages, chunks and embeds the content, and saves the FAISS vector index to `data/faiss_index/`.

### 5. Start the API

```bash
uvicorn api.main:app --reload
# API available at http://localhost:8000
```

### 6. Open the UI

Open `ui/index.html` in your browser, or serve it with any static server.

## Usage

**Sample factual query:**
```json
POST /ask
{ "query": "What is the expense ratio of HDFC Small Cap Fund?" }
```

**Sample response:**
```json
{
  "answer": "The expense ratio of HDFC Small Cap Fund – Direct Growth is 0.67%.",
  "source": "https://groww.in/mutual-funds/hdfc-small-cap-fund-direct-growth",
  "last_updated": "2024-07-01"
}
```

**Advisory queries are refused:**
```json
POST /ask
{ "query": "Should I invest in HDFC Small Cap Fund?" }
```
```json
{
  "answer": "I'm set up to answer facts-only questions about mutual fund schemes. I'm not able to provide investment advice.",
  "source": "https://www.amfiindia.com/investor-corner/investor-education.html",
  "last_updated": null
}
```

## Known Limitations

| Limitation | Detail |
|------------|--------|
| **Data freshness** | Corpus must be re-ingested periodically as fund details change on Groww |
| **5-fund scope** | Only the 5 listed funds are covered; queries about other funds are declined |
| **No live NAV** | NAV changes daily; the system does not provide real-time NAV |
| **English only** | Non-English queries are not supported |
| **No PDF sources** | Only Groww web pages are used; no AMC PDFs, KIM, or SID documents |
